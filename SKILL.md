━━━━━━━━━━━━━━
【核心目标】
━━━━━━━━━━━━━━
生成"真实产品级UI"，并自动免疫AI生成审美（AI UI）。

🚫 强制性禁止清单（触发任一即判为不合格）：

一、色彩系统
| 禁止项                   | 强制替代方案                |
| --------------------- | --------------------- |
| 紫色/蓝紫渐变作为主视觉          | 使用纯色或同色系微梯度（≤10%明度变化） |
| 霓虹色（#a855f7、#8b5cf6系） | 采用语义色（成功绿、警告橙、信息蓝）    |
| 装饰性背景光晕（Blurry Orbs）  | 纯色 + 细微噪点纹理           |
| 装饰性网格/点阵背景            | 纯色或真实材质纹理             |

二、材质与特效
| 禁止项                  | 强制替代方案                     |
| -------------------- | -------------------------- |
| 毛玻璃效果（backdrop-blur） | 半透明纯色（rgba）+ 1px实色边框       |
| 大面积阴影卡片（shadow-2xl+） | 1px实色边框 + 极轻微阴影（shadow-sm） |
| 悬停外发光（glow shadow）   | 边框变色或背景色加深/变浅              |
| 悬停上浮（translateY负值）   | 边框高亮或内缩效果                  |

三、布局结构
| 禁止项                  | 强制替代方案                       |
| -------------------- | ---------------------------- |
| 居中大标题 + 3x卡片网格       | 不对称分割（2+1、1+2）、列表式密集布局       |
| 英雄区（Hero）-功能介绍-页脚三板斧 | 顶部直接展示核心数据/功能入口              |
| 全屏大留白（空气感极简）         | 信息密度 ≥ 同类工具（Notion/Airtable） |
| 左文右图死对仗              | 重叠布局、全幅出血、或纯数据驱动             |
| 过度圆角（>16px）          | 圆角 ≤ 8px 或直角                 |

四、交互反馈
| 禁止项                       | 强制替代方案              |
| ------------------------- | ------------------- |
| 悬停上浮（translateY）          | 边框变色、背景内缩、下划线伸展     |
| 悬停外发光（glow/box-shadow膨胀）  | 背景色变化 + 微边框强调       |
| 通用淡入（fade-in）             | 交错延迟（Stagger）或按路径移动 |
| 弹性缓动（bouncy cubic-bezier） | ease 缓动（150-200ms）  |

五、字体与文案
| 禁止项                    | 强制替代方案                   |
| ---------------------- | ------------------------ |
| Inter / Roboto / Arial | 系统默认或 Sora / Outfit（标题用） |
| 全大写排版（ALL CAPS）        | 常规大小写                    |
| "赋能"、"无缝"、"解锁"、"革新"    | 直白动词（"记账"、"筛选"、"导出"）     |
| Emoji作为唯一图标            | SVG图标或CSS几何图形            |

✅ 必须符合的设计规范：
- 风格参照：Notion / Airtable / Linear / Figma / ClickUp
- 视觉基调：system-ui / enterprise software / 工具系统
- 信息密度：高密度 + 强功能导向
- 顶部必须有导航栏或工具栏，禁止"纯海报式"首页
- 卡片必须有明确边界（边框优先于阴影）
- 主色：单一品牌色（非紫色），用于关键操作按钮
- 中性色：浅灰、中灰、深灰
- 语义色：绿（成功）、红（警告）、橙（待处理）、蓝（信息）

【Hard Constraints（绝对约束）】
━━━━━━━━━━━━━━

1. 必须先生成 phase_1
2. 必须完成 phase_2 评分，Critic 必须引用具体代码行
3. 如果 score > 25，必须执行 phase_3，最多重试3次
4. 如果3次重试后 score 仍 > 25，在 final_output 中标注 critic_final_verdict = "FAIL"
5. 最终只能输出 final_output
6. 禁止输出任何解释、分析、思考过程
7. 输出必须是合法 JSON

━━━━━━━━━━━━━━
【评分标准（量化扣分制）】
━━━━━━━━━━━━━━
基础分 = 0（假设是完美UI）

| 检测项                                      | 扣分   |
| ---------------------------------------- | ---- |
| 检测到紫色/蓝紫渐变（background中的gradient且色相在240-300°） | +20  |
| 检测到 backdrop-blur 属性                     | +15  |
| 检测到 box-shadow 的 blur > 20px             | +10  |
| 检测到卡片圆角 > 16px（border-radius: 大于16px）    | +10  |
| 检测到 hover 中的 translateY（向上浮动）            | +10  |
| 检测到 "Empower"/"无缝"/"解锁"/"革新" 文案          | +10  |
| 检测到 3个以上同级卡片且宽度相等（3x网格）                  | +15  |
| 检测到 Inter / Roboto / Arial 字体指定          | +10  |
| 检测到页面无表格/列表/数据行（信息密度低）                   | +10  |

👉 总分为各项累计，≥25即为不合格，必须进入 phase_3。

━━━━━━━━━━━━━━
【反绕过机制（Anti-Cheat）】
━━━━━━━━━━━━━━
为防止 AI 故意打低分逃避修复，Critic 必须遵守：
1. 禁止给出低于10分的评分，除非你能逐行证明每一处都用了实线边框且无任何阴影
2. 如果评分 < 10，你必须在 issues 中列出所有检查过的代码行范围
3. 如果上一轮评分 < 25 但 final_output 中仍有明显 AI 风格，视为作弊，重新触发 phase_3
